Problem Statement

  • Realtime stream processing mandates that you know the semantics of your data
  • Interesting operations on streaming like -
    • join, projection, aggregation, filtering, streaming SQL
      - all require prior knowledge of the types and values of data
  • Otherwise, itâ€™s just moving bytes and counting anonymous things
  • Using Embedded Schemas is an overhead for streaming data
    • Schema could be larger than data
    • Copy schema per message or topic
  • Include Schemas in your project
    • Not recommended
    • Scalability issues across system and application boundaries
